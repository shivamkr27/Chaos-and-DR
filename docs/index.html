RECOVERY CONTROLS
WHAT THIS PROJECT IS
Chaos Engineering + Disaster Recovery on AWS. App runs on Primary (us-east-1). On failure, Cloudflare Worker automatically routes traffic to DR (us-west-2) in under 5 seconds — no DNS change, no TTL wait. Chaos experiments validate recovery under real failure conditions.
ARCHITECTURE
  ┌─────────────────────────────────────┐
  │        Cloudflare Worker            │
  │  Probes /health/live per request    │
  │  Routes to DR if primary is DOWN    │
  └───────┬──────────────────┬──────────┘
          │ normal           │ failover (<5s)
  ┌───────▼──────┐   ┌───────▼──────────┐
  │  PRIMARY      │   │  DR               │
  │  us-east-1    │   │  us-west-2        │
  │  EC2 t3.micro │   │  EC2 t3.micro     │
  │  K3s 2 pods   │   │  K3s 1 pod        │
  │  Prometheus   │   │  Prometheus       │
  │  Grafana      │   │  (standby)        │
  │  RDS Postgres │   │  no DB — degraded │
  └──────────────┘   └───────────────────┘
STACK
Node.js → Docker → K3s (Kubernetes) → EC2 t3.micro → AWS
Prometheus + Grafana · Cloudflare Worker (failover)
HPA autoscaling/v2 · GitHub Actions CI/CD · AWS SNS + Lambda alerts
